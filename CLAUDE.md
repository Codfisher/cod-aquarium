# 專案概述

此專案為鱈魚（codlin）的個人部落格

## Coding Style

- 變數名稱禁止使用縮，除非是 url 這種廣為人知的縮寫
- 矩陣資料用 list 結尾。例如：Hole[]，應該命名為 holeList 而不是 holes。物件內多個參數才用複數結尾，例如 Vue 的 props、attrs
  此目的是為了與 Vue 本身的命名規則保持相容，避免混淆。
- Map、Set 資料用 map、set 結尾。例如：Map<string, Hole>，應該命名為 holeMap 而不是 holesMap。

- **TypeScript**:
  - 禁止使用 `any`，必須定義明確的 Interface 或 Type。

## AI 行為準則 (Instructions for AI)

當你生成程式碼或回答問題時，請遵守：

1. **回應語言**：除非另有指示，否則請使用正體中文，且結尾統一加上 `🐟` 符號。
2. **Commit Message**：請使用中文，並遵循 [Conventional Commits](https://www.conventionalcommits.org/zh-hant/v1.0.0/) 的格式。

## 寫作風格規則（Writing Style）

協助撰寫部落格文章時，請遵守以下規則。

### 身份定位

- 作者自稱「鱈魚」，簽名符號為 `🐟`
- 開場慣用語：「大家好，我是鱈魚」

### 語氣

- 親切隨意，非正式口吻，避免學術或書面語氣
- 禁止使用「筆者」等書面自稱
- 口頭禪：「其實」、「不過」、「雖然說」、「的確」、「重點是」
- 可適度自嘲，例如：「我就老，我就是要用」
- 讚美用語：「超酷超強」、「超級有趣」、「真的很不錯」

### 讀者互動

- 稱呼讀者：「大家」、「各位」、「各位大大」
- 使用虛擬「路人」角色進行問答，代表讀者的疑問：

  ```text
  路人：「為什麼要這樣做？」
  鱈魚：「因為...」
  ```

- 固定結尾語：「感謝您讀到這裡，如果您覺得有收穫，歡迎分享出去。」
- 謙遜語：「有錯誤還請多多指教」

### 顏文字

- **大量使用顏文字**（非現代 emoji），這是個人品牌的核心，不可省略
- 常用顏文字：`( •̀ ω •́ )✧`、`ᕕ( ﾟ ∀。)ᕗ`、`ԅ(´∀` ԅ)`、`◝(≧∀≦)◟`
- 顏文字需用 `%%` 包裹，防止折行。例：`%( •̀ ω •́ )✧%`
- 段落結尾常搭配顏文字，取代或補充句點

### 標題格式

- H1：問句或陳述句，常帶顏文字。例：「要不要來點 Side Project 啊？ԅ(´∀` ԅ)」
- H2：前綴 emoji 作視覺標記。例：「💭 為什麼要做 Side Project？」、「🧰 使用方法」
- H3：名詞短語，直接具體

### 文章結構

固定格式：

1. **開場**：背景故事、痛點描述、或直接提問
2. **正文**：分段說明（每段 2-4 句），穿插代碼範例
3. **結尾**：固定「總結 🐟」段落，用 `-` 列出 3-5 個核心要點，最後加感謝語

### 中英混用

- 技術術語保留英文原文：API、Promise、Vue Router、memoization、Computed、Watcher
- 專有名詞保留：Sentry、Pinia、VitePress、Quasar
- 可創意中文化：「打工仔」= Web Worker
- 首次出現技術詞彙，視情況補充中文說明

### 段落與代碼

- 段落短，平均 2-4 句，閱讀節奏輕快
- 代碼塊前後必有文字說明
- 代碼語言標記：`ts`、`js`、`vue`、`json`、`bash`
- 可用 `[!code ++]`、`[!code --]` 標記新增/刪除行
