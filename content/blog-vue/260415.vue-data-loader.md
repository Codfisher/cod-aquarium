# 用 Vue Data Loader 解決 fetch 瀑布流問題

大家好，我是鱈魚。(｡･∀･)ﾉﾞ

在 Vue 專案中，你有沒有遇過這種情境：頁面需要顯示使用者資料、文章列表和留言，結果 API 一個接一個地等，畫面一塊一塊地冒出來？

這就是所謂的 **fetch 瀑布流（Waterfall）** 問題。

## 什麼是 fetch 瀑布流？

先來看一個常見場景。頁面有三層巢狀元件，各自在 `onMounted` 取資料：

```vue
<!-- UserPage.vue -->
<script setup lang="ts">
import { ref, onMounted } from 'vue'

const user = ref()

onMounted(async () => {
  user.value = await fetchUser(1)    // 等 1 秒
})
</script>

<template>
  <div v-if="user">
    <h1>{{ user.name }}</h1>
    <UserPostList :user-id="user.id" />
  </div>
</template>
```

```vue
<!-- UserPostList.vue -->
<script setup lang="ts">
import { ref, onMounted } from 'vue'

const postList = ref([])

onMounted(async () => {
  postList.value = await fetchPostList(props.userId)  // 再等 1 秒
})
</script>

<template>
  <ul v-if="postList.length">
    <li v-for="post in postList" :key="post.id">
      {{ post.title }}
    </li>
    <PostCommentList :post-id="postList[0].id" />
  </ul>
</template>
```

```vue
<!-- PostCommentList.vue -->
<script setup lang="ts">
import { ref, onMounted } from 'vue'

const commentList = ref([])

onMounted(async () => {
  commentList.value = await fetchCommentList(props.postId)  // 又等 1 秒
})
</script>
```

每支 API 都只需要 1 秒，但因為元件要等父元件 fetch 完才會 render，子元件的 `onMounted` 才會觸發，形成瀑布流：

```
User     ████████░░░░░░░░░░░░░░░░  1s
Post     ░░░░░░░░████████░░░░░░░░  1s
Comment  ░░░░░░░░░░░░░░░░████████  1s
                            總耗時：3s
```

三個各 1 秒的 API，加總卻花了 3 秒。%( ´•̥̥̥ω•̥̥̥` )%

## 常見的解法

最直覺的解法是在最上層用 `Promise.all` 一次打完：

```ts
const [user, postList, commentList] = await Promise.all([
  fetchUser(1),
  fetchPostList(1),
  fetchCommentList(1),
])
```

這的確能解決瀑布流，但帶來新問題：

- **資料取得邏輯與元件脫鉤**，本來各元件自己管自己的資料，現在全部集中在父元件
- **props 層層傳遞**，子元件不再自己取資料，改由父元件透過 props 餵入
- **元件重用困難**，換個頁面用同一個元件，又要重新處理資料取得邏輯

有沒有辦法既保持元件各自管理資料，又能平行取得呢？

## Vue Data Loader

[Vue Data Loader](https://uvr.esm.is/data-loaders/) 是 `unplugin-vue-router` 提供的功能，核心概念是把資料取得邏輯**定義在路由頁面元件上，由路由器在導航時統一觸發**。

簡單來說就是：導航發生時，路由器會收集該頁面所有的 loader 並**平行執行**，全部完成後才進入頁面。

使用方式如下：

```vue
<script lang="ts">
import { defineBasicLoader } from 'unplugin-vue-router/data-loaders/basic'

export const useUserLoader = defineBasicLoader('/user', async () => {
  return fetchUser(1)
})

export const usePostListLoader = defineBasicLoader('/user', async () => {
  return fetchPostList(1)
})

export const useCommentListLoader = defineBasicLoader('/user', async () => {
  return fetchCommentList(1)
})
</script>

<script setup lang="ts">
const { data: user } = useUserLoader()
const { data: postList } = usePostListLoader()
const { data: commentList } = useCommentListLoader()
</script>
```

三個 loader 在導航時平行啟動：

```
User     ████████░░░░░░░░░░░░░░░░  1s
Post     ████████░░░░░░░░░░░░░░░░  1s
Comment  ████████░░░░░░░░░░░░░░░░  1s
                            總耗時：1s
```

同樣三支 API，從 3 秒縮短到 1 秒。%( •̀ ω •́ )✧%

::: tip defineBasicLoader 為什麼要放在非 setup 的 `<script>` 中？
因為 loader 需要在**模組層級**被定義，路由器才能在導航階段（元件尚未建立）就偵測並執行它。

放在 `<script setup>` 裡的程式碼屬於元件實例層級，必須等元件建立後才執行，那就又回到 `onMounted` 的老問題了。
:::

## 總結 🐟

- 元件各自在 `onMounted` 取資料，層層嵌套會形成 fetch 瀑布流，拖慢頁面載入
- `Promise.all` 可以解決，但會讓資料取得邏輯集中在父元件，犧牲元件的獨立性
- Vue Data Loader 讓每個路由頁面定義 loader，路由器在導航時平行執行，兼顧效能與程式碼組織
- `defineBasicLoader` 要放在非 setup 的 `<script>` 中，才能在導航階段被路由器偵測
