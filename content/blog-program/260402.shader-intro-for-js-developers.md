## Shader 是什麼？

先來看大局。當我們在瀏覽器畫出一個畫面時，背後其實有一條「渲染管線」在運作。

Shader 就是這條管線裡，**你可以自己寫程式控制**的兩個關鍵步驟。

<DiagramPipeline />

重點來了：

- **Vertex Shader**：決定每個頂點的位置（形狀長在哪裡）
- **Fragment Shader**：決定每個像素的顏色（畫面長什麼樣子）

今天我們會先專注在 **Fragment Shader**，因為它最直覺，改了馬上看到顏色變化，超有成就感。

## GPU 與 CPU 的思維差異

在學 Shader 之前，有一個關鍵的腦袋轉換要先做好。

用 JS 寫程式時，我們習慣的是 CPU 的思考方式：「一步一步，一個一個來」。

但 GPU 不是這樣運作的，GPU 是「一口氣處理所有像素」。

<DiagramCpuGpu />

實際跑起來差異有多大？看看下面這隻像素鱈魚就知道了。

左邊 CPU 要一格一格慢慢塗，右邊 GPU 幾乎一瞬間就全部亮起來。%( •̀ ω •́ )✧%

<DemoCpuGpu />

這代表什麼？

**你寫的 Fragment Shader 程式碼，會在每個像素上「同時」執行一次。**

所以你不能像 JS 那樣說「幫我把第 100 個 pixel 塗成紅色」，而是要說「如果我現在所在的位置在某個範圍內，就塗成紅色」。

<br>

路人：「聽起來像是每個 pixel 都有自己的小腦袋在思考？」

鱈魚：「對！而且它們之間完全不能互相聊天，是平行宇宙的概念。%(•̀ω•́)✧%」

## GLSL 語法速成 — JS 開發者對照表

Shader 使用的語言叫 **GLSL**（OpenGL Shading Language）。別被名字嚇到，它長得跟 C 很像，而且語法比 JS 還簡單。

### 型別對照

| JS / TS | GLSL | 說明 |
|---------|------|------|
| `number` | `float` | 浮點數（GLSL 沒有整數混用，`1` 要寫成 `1.0`） |
| `number` | `int` | 整數 |
| `boolean` | `bool` | 布林值 |
| — | `vec2` | 二維向量 `vec2(x, y)` |
| — | `vec3` | 三維向量 `vec3(r, g, b)` 或 `vec3(x, y, z)` |
| — | `vec4` | 四維向量 `vec4(r, g, b, a)` |

### 向量操作 — GLSL 的超能力

向量是 GLSL 最強大的特色之一，它可以用一行做到 JS 需要好幾行才能完成的事。

```glsl
vec3 color = vec3(1.0, 0.5, 0.3);

// Swizzling：隨意取出分量重組
vec2 rg = color.rg;     // vec2(1.0, 0.5)
vec3 bgr = color.bgr;   // vec3(0.3, 0.5, 1.0)

// 向量運算：整組一起算
vec3 a = vec3(1.0, 2.0, 3.0);
vec3 b = vec3(4.0, 5.0, 6.0);
vec3 c = a + b;  // vec3(5.0, 7.0, 9.0)
vec3 d = a * 2.0; // vec3(2.0, 4.0, 6.0)
```

同樣的邏輯用 JS 來寫的話：

```js
const color = [1.0, 0.5, 0.3]

// 取出分量要自己 index
const rg = [color[0], color[1]]
const bgr = [color[2], color[1], color[0]]

// 向量運算要用迴圈或 map
const a = [1.0, 2.0, 3.0]
const b = [4.0, 5.0, 6.0]
const c = a.map((value, i) => value + b[i])
const d = a.map((value) => value * 2.0)
```

路人：「好像也沒那麼恐怖？」

鱈魚：「對吧！GLSL 反而更精簡，因為它天生就是為向量運算設計的。%(●'◡'●)%」

### 常用內建函式

| GLSL 函式 | 對應概念 | 說明 |
|-----------|---------|------|
| `mix(a, b, t)` | 線性插值 | `a * (1-t) + b * t`，`t` 在 0~1 之間 |
| `step(edge, x)` | 階梯函式 | `x < edge ? 0.0 : 1.0` |
| `smoothstep(a, b, x)` | 平滑階梯 | 在 a~b 之間平滑過渡 |
| `length(v)` | 向量長度 | `sqrt(v.x² + v.y²)` |
| `clamp(x, min, max)` | 限制範圍 | 等同 `Math.min(Math.max(x, min), max)` |
| `sin(x)` / `cos(x)` | 三角函式 | 和 JS 的 `Math.sin` 一樣 |
| `abs(x)` | 絕對值 | 和 JS 的 `Math.abs` 一樣 |

## Fragment Shader：你的第一個像素函式

好，理論講夠了，讓我們來動手！

Fragment Shader 的核心概念超級簡單：

<DiagramFragment />

每個 pixel 都會問自己兩個問題：

1. **我在哪裡？** → `gl_FragCoord.xy`（像素座標）
2. **我要什麼顏色？** → `gl_FragColor = vec4(r, g, b, a)`

就是這樣！整個 Fragment Shader 說穿了就是一個函式：「給定座標，回傳顏色」。

<br>

來試試你的第一個 Shader，下方的編輯器可以直接修改程式碼，改完會即時更新畫面。

試著把 `vec4(0.2, 0.6, 1.0, 1.0)` 裡的數字改成其他值看看？%(ˊᗜˋ*)%

<ShaderPlayground :initial-code="PRESET_SOLID_COLOR.code" />

::: tip 小提醒
GLSL 的顏色值範圍是 `0.0` ~ `1.0`，不是 CSS 的 `0` ~ `255`。所以 `vec4(1.0, 0.0, 0.0, 1.0)` 就是純紅色。
:::

### 座標正規化 — UV 的概念

直接用 `gl_FragCoord` 的話，座標會是實際的像素值（例如 800, 600），不太方便計算。

所以通常我們會把座標「正規化」到 `0.0 ~ 1.0` 的範圍，這個正規化後的座標叫做 **UV**。

<DiagramUv />

```glsl
vec2 uv = gl_FragCoord.xy / u_resolution;
```

這樣不管畫布多大，`uv.x` 和 `uv.y` 都會在 0 到 1 之間，超方便。

來看看用 UV 座標做出漸層效果有多簡單。

<ShaderPlayground :initial-code="PRESET_GRADIENT.code" />

是不是很神奇？只是把座標值當成顏色值，就自動產生了漸層。%(σﾟ∀ﾟ)σ%

## Uniform：JS 與 Shader 的溝通橋樑

到目前為止，Shader 都只是自己跟自己玩。但實際上我們常常需要從 JS 傳資料進去，例如「現在過了幾秒」或「滑鼠在哪裡」。

這時候就要用 **Uniform**。

<DiagramUniform />

`uniform` 這個字在 GLSL 裡是一個修飾詞，代表「這個值從外部（JS）傳進來，而且**每個 pixel 都拿到同樣的值**」。

本文的互動編輯器已經幫你注入了三個常用的 uniform：

| Uniform | 型別 | 說明 |
|---------|------|------|
| `u_time` | `float` | 從啟動開始經過的秒數 |
| `u_resolution` | `vec2` | 畫布寬高（像素） |
| `u_mouse` | `vec2` | 滑鼠在畫布上的位置 |

來試試用 `u_time` 做動畫效果。

### 畫圓的原理

畫圓其實就是一道數學題：**判斷目前的 pixel 距離圓心有多遠**。

<DiagramCircle />

用到的關鍵函式：

- `length(v)`：計算向量長度（就是畢氏定理）
- `step(edge, x)`：硬邊界切換
- `smoothstep(a, b, x)`：柔和邊界
- `mix(colorA, colorB, t)`：混色

### 挑戰時間

試試看能不能改出這些效果：

1. 把圓形變成橢圓（提示：對 UV 的 x 或 y 乘以不同倍數）
2. 讓圓形跟著時間移動（提示：圓心加上 `sin(u_time)` 位移）
3. 畫出同心圓（提示：對 distance 做 `sin` 或 `fract`）

## 總結 🐟

- Shader 是在 GPU 上執行的小程式，分為 **Vertex Shader**（處理位置）和 **Fragment Shader**（處理顏色）
- GPU 的思維是「每個 pixel 同時獨立計算」，不是 CPU 的「一步一步來」
- GLSL 語法簡潔，核心就是向量運算，對 JS 開發者來說門檻沒想像中高
- Fragment Shader 的本質就是一個函式：「給我座標，我回傳顏色」
- 透過 **Uniform** 可以從 JS 傳入時間、滑鼠位置等動態資料
