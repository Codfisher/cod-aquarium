# BeyBattle 戰鬥階段動漫演出系統設計

## 概述

為戰鬥階段加入漸進式升溫的動漫風演出系統，讓戰鬥從平穩開場逐步推向戲劇性高潮。

## 漸進式視覺升溫

由「雙方最低轉速百分比」驅動三個演出等級，每個階段疊加上一階段效果：

### 平穩期（最低轉速 > 60%）
- 碰撞：小型火花 + 輕微震動
- 陀螺：正常旋轉，微弱移動軌跡線

### 激烈期（最低轉速 30%~60%）
- 碰撞：中型衝擊波環 + 漫畫放射線條 + 更多火花
- 場地：碗面能量波紋（碰撞點向外擴散同心圓）
- 陀螺：高速移動時殘影拖尾（3~5 個半透明殘影）
- 全域：畫面四角漸暈加深

### 決戰期（最低轉速 < 30%）
- 碰撞：全屏衝擊波 + 慢動作 0.4 秒 + 鏡頭拉近碰撞點再拉回
- 場地：碗面邊緣發光脈衝，底部光源閃爍
- 陀螺：低轉速方紅色危險光暈
- 全域：紅色脈衝 vignette + 持續速度線背景

## 戲劇性事件系統

| 事件 | 觸發條件 | 演出 |
|------|---------|------|
| 首次碰撞 | 第一次碰撞 | 0.3 秒慢動作 + 震動 + 「CLASH!」文字 |
| 連擊 Combo | 1.5 秒內碰撞 ≥3 次 | 閃白 + Combo 計數（×3, ×4...）+ 火花加倍 |
| 懸崖邊緣 | 離出界 < 半徑 10% | 特寫鏡頭 + 紅色警告框 + 「DANGER!」 |
| 絕地反擊 | 轉速 < 15% 成功擊退對手 | 0.5 秒慢動作 + 金色爆發光 + 「COUNTER!」 |
| 最後一擊 | 造成出界或停轉 | 1 秒超慢動作 + 追蹤鏡頭 → 全景 + 全屏衝擊波 |

### 優先級
最後一擊 > 絕地反擊 > 懸崖邊緣 > 連擊 > 首次碰撞

### 冷卻
- 懸崖邊緣、Combo：3 秒冷卻
- 首次碰撞：一次性
- 最後一擊、絕地反擊：無冷卻

## 視覺元素實作方式

### 速度線
- 決戰期持續繪製從四周向中心匯聚的白色線條（CSS overlay）
- 強度隨最低轉速遞增

### 碰撞衝擊波
- Babylon.js CreateDisc 快速放大 + 淡出（0.3 秒）
- 激烈期：1 層白色半透明環
- 決戰期：3 層環疊加（白 → 黃 → 橘）+ 放射線條

### 殘影拖尾
- 每 3 幀記錄位置，渲染 3~5 個遞減 opacity 複製體
- 只在移動速度 > 閾值時顯示

### 事件文字
- Orbitron 字體，放大進入 + 向上淡出，持續 0.8 秒
- 不阻擋戰鬥進行

### 鏡頭演出
- 慢動作：物理 deltaTime × 縮放係數（0.2），render loop 正常
- 拉近：camera.radius 平滑過渡
- 追蹤：camera.target 臨時指向碰撞點，0.5 秒後回歸
