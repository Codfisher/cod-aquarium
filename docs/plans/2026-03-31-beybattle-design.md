# BeyBattle — 戰鬥陀螺 3D 網頁遊戲設計

## 概述

一款基於 Babylon.js 的 3D 戰鬥陀螺網頁遊戲。玩家可自由組裝陀螺零件、透過多段 QTE 發射陀螺，在碗狀競技場中與 AI 對戰。卡通風格渲染，支援桌面與觸控裝置。

## 決策紀錄

| 項目 | 決策 | 原因 |
|------|------|------|
| 對戰模式 | 單人 vs AI（預留多人擴展） | 先做核心體驗 |
| 配置系統 | 零件組裝制（三層） | 真實戰鬥陀螺的玩法 |
| 場地 | 經典圓形碗狀（之後擴展） | 簡單直覺 |
| QTE | 三階段（蓄力、瞄準、發射） | 有層次感的操作體驗 |
| 視角 | 俯視 45° 固定鏡頭 | 乾淨俐落 |
| 勝負 | 混合制（出界 or 轉速歸零） | 增加戰鬥多樣性 |
| 美術 | 卡通風格（cel-shading） | 適合熱血主題 |
| 建模 | 純程式化生成 | 不需外部資源，開發快，之後可升級 GLB |
| 物理 | 自寫簡化 2D 物理 | 碗面運動本質 2D，不需完整物理引擎 |

## 陀螺零件系統

陀螺由三層零件組成，各自影響四項屬性：

### 屬性定義

```typescript
interface BeybladeStats {
  attack: number;   // 碰撞擊飛力
  defense: number;  // 抵抗擊飛
  stamina: number;  // 轉速衰減率（越高衰減越慢）
  speed: number;    // 移動速率
}
```

### 攻擊環（Attack Ring）— 最上層

| 名稱 | 造型 | 效果 |
|------|------|------|
| 鋸齒環 | 圓盤邊緣鋸齒突起 | 高攻擊力，擊飛力強 |
| 圓滑環 | 平滑圓盤 | 低攻擊，摩擦低不易被反彈 |
| 三角環 | 三個大型三角突起 | 極高單次擊飛，間隔大易落空 |
| 六翼環 | 六片對稱翼片 | 平衡攻擊，空氣阻力減速 |

### 重量盤（Weight Disk）— 中層

| 名稱 | 造型 | 效果 |
|------|------|------|
| 重力盤 | 外圈加厚寬盤 | 極難擊飛，轉速衰減快 |
| 輕量盤 | 薄型小直徑 | 移動快，易被推動 |
| 平衡盤 | 均勻厚度中型 | 各項平衡 |
| 偏心盤 | 重心偏移異形 | 移動軌跡不規則 |

### 軸心（Spin Tip）— 最下層

| 名稱 | 造型 | 效果 |
|------|------|------|
| 尖銳軸 | 細長圓錐 | 最低摩擦，衰減慢，移動範圍大 |
| 平坦軸 | 扁平底部 | 高摩擦，幾乎不移動，衰減快 |
| 球形軸 | 圓球底部 | 中等摩擦，移動靈活 |
| 彈簧軸 | 可壓縮彈簧 | 吸收碰撞衝擊，不易出界 |

三個零件屬性加總為陀螺最終屬性。

## QTE 發射系統

三階段 QTE，全程支援鍵盤（空白鍵）與觸控（點擊螢幕任意處）。

### 階段一：蓄力（Charge）→ 影響初始轉速

- 垂直能量條，指標上下來回移動
- 頂部 15% 為 Perfect（100% 轉速）、外圍各 10% 為 Great（80%）、其餘 Good（60%）
- 時限 3 秒，超時自動判定

### 階段二：瞄準（Aim）→ 影響落點位置

- 俯視場地，十字準心沿圓形軌跡移動
- 點擊決定落點，無評級，純策略選擇
- 攻擊型靠近對手、防禦型佔中心、持久型可選邊緣

### 階段三：發射（Launch）→ 影響初始移動方向

- 從落點出現旋轉方向箭頭
- 點擊決定發射方向
- 可選直衝、繞圈、朝中心

### AI 的 QTE 結果

| 難度 | 轉速倍率範圍 |
|------|------------|
| 簡單 | 0.6 ~ 0.8 |
| 中等 | 0.75 ~ 0.9 |
| 困難 | 0.85 ~ 1.0 |

## 物理模擬

### 運動模型

```typescript
interface BeybladeState {
  position: Vector2;      // 場地 2D 位置
  velocity: Vector2;      // 速度向量
  spinRate: number;       // 轉速（RPM）
  rotationAngle: number;  // 視覺旋轉角度
}
```

### 每幀更新

1. **碗面重力**：離中心越遠向心力越強
2. **摩擦力**：依軸心類型的摩擦係數減速
3. **轉速衰減**：依 stamina 自然衰減，轉速越低移動越慢
4. **速度上限**：由 speed 屬性決定

### 碰撞系統

- 碰撞體：圓形（半徑由攻擊環決定）
- 擊飛力 = 攻擊方 `attack × (spinRate / maxSpinRate)`
- 抵抗力 = 防禦方 `defense × (weight / maxWeight)`
- 實際擊退 = `max(擊飛力 - 抵抗力, 最小值)`
- 碰撞方向：兩陀螺中心連線法線
- 碰撞時雙方轉速各損失小百分比

### 勝負判定

- **出界**：陀螺中心超出場地邊緣半徑 → 立刻敗北
- **轉速歸零**：spinRate ≤ 0 → 敗北

## 視覺風格

### 卡通渲染

- Babylon.js CellMaterial 或自訂 shader，光照離散化 2~3 色階
- 後處理邊緣偵測描邊（depth + normal）
- 陀螺：鮮豔飽和色（紅、藍、金、紫）
- 場地：低飽和灰白，襯托陀螺

### 特效

- 碰撞：星形火花粒子 + 放射線條 + 畫面微震
- 強力碰撞：慢動作 0.3 秒
- QTE Perfect：畫面閃白 + 文字跳出動畫

### 競技場

- 程式化生成碗狀旋轉體（lathe）
- 內壁淺色格線紋理
- 邊緣裝飾外框
- 半透明，底部微弱光暈

## AI 系統

AI 不直接操控物理，強弱來自配置策略與 QTE 結果。

### 零件選擇策略

| 難度 | 策略 |
|------|------|
| 簡單 | 隨機 |
| 中等 | 偏向單一流派（純攻/純防/純持久） |
| 困難 | 針對玩家配置反制 |

- 按鈕最小尺寸 44px
- 配置畫面：卡片式滑動選擇
